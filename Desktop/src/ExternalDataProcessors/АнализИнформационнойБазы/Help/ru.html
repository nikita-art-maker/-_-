Обработка "Анализ количества данных"

Обработка "Анализ количества данных" предназначена для экспресс-анализа количества записей различных таблиц информационной базы. Обработка может функционировать как в обычном приложении, так и в управляемом приложении.

Вид анализируемых обработкой данных устанавливается с помощью флажков в группе "Виды анализируемых данных". Если информационная база не содержит объектов какого-либо вида, соответствующий флажок в группе сброшен и недоступен для изменения.

Для периодических объектов информационной базы (документов, периодических регистров сведений, регистров накопления, регистров бухгалтерии и регистров расчета) можно задать дополнительные параметры анализа в группе "Параметры периодического анализа":

  • Период с - дата начала анализируемого периода. Если параметр не задан, начальный период не используется;
  • По - дата окончания анализируемого периода. Если параметр не задан, конечный период не используется;
  • Периодичность - периодичность группировки записей по дате. Может принимать значения День, Месяц, Квартал, Год. Если периодичность не задана, группировка по дате не используется.

После задания видов анализируемых данных и параметров периодического анализа необходимо нажать кнопку "Выполнить". Обработка выполнить анализ количества данных в соответствии с заданными условиями и результать анализа выведет в табличное поле, расположенное в нижней части формы.

Анализируемые данные выводятся в виде дерева, состоящего из двух колонок "Имя" и "Кол-во". В колонке "Имя" выводится имя анализируемых объектов, а в колонке "Кол-во" выводится количество соответствующих объектов. Дерево анализа упорядочено по убыванию количества объектов соответствующего вида. Объекты информационной базы с нулевым количеством не выводятся.

Для непериодических объектов (справочников, планов видов характеристик, планов счетов, непериодических регистров сведений, бизнес-процессов и задач) выводится количество записей в пределах всей информационной базы. Для периодических объектов (документов, периодических регистров сведений, регистров накопления, регистров бухгалтерии и регистров расчета), если не задан период анализа, также выводится количество записей в пределах всей информационной базы, иначе - выводится количество записей в пределах анализируемого периода. Для периодических объектов, если задана периодичность, количество записей группируется в пределах периодов. Периоды указываются для каждого объекта в хронологическом порядке.

Дерево анализа можно вывести на печать с помощью контекстной команды "Вывести список".
